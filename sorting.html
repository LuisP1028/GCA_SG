Part III: Sorting

1. Partitioning & Grouping
Partitioning with Many Duplicates

Rearrange the list so that all items sharing the same key sit next to each other using Cyclic Sort / Routing.

  1. Frequency Pass: Build a map (Key â†’ Count) to virtually "reserve" blocks.
  2. Offset Map: Convert counts into absolute Start Indices (Write Heads).
  3. Routing Loop: Pick a group. Inspect the intruder at the Write Head. Swap the intruder to its correct home. When correct, advance the Write Head pointer.
Removing Duplicates In-Place

Filter an array without allocating new memory using the Two Pointer Technique.

  1. Lexicographical Grouping: Sort so duplicates are neighbors.
  2. Write Pointer: Initialize at index 1.
  3. Scan: If A[read] != A[write-1], overwrite A[write] and increment write pointer.
  4. Truncation: Delete the obsolete buffer at the end.
2. Interval Management
Union of Intervals (Sort & Sweep)

Convert a complex comparison problem into a linear stream of events.

  • Sort: By Left Endpoint. If equal, put Closed start before Open start.
  • Sweep: Maintain one Active Interval.
  • Gap: If Next starts strictly after Active ends, save Active. Next becomes Active.
  • Merge: If overlap, absorb Next. New End = max(Active.end, Next.end).
Merging Intervals (Event Based)

Phase 1 (Left): Skip non-overlapping. Right < New Left.

Phase 2 (Merge): Overlap zone. New Left = min, New Right = max. Mutate the temporary new_interval.

Phase 3 (Flush): Append the merged interval, then append the remainder.

Interval Overlap / Concurrency

Find the peak number of events active at a single moment.

The How: Create a timeline of markers (Start/End) and sort them. Tie Breaker: Process START before END to count overlaps correctly.

Sweep: +1 for Start, -1 for End. Track the highest value reached.

3. Merging & Constructive
Merge Two Sorted Arrays

Strategy: Three Pointers (Tail A, Tail B, Write).

The Trick: Start filling from the back (end of the buffer) to avoid shifting. Compare A[tail] vs B[tail], place the larger one at A[write], and decrement pointers.

Set Intersection

Find common elements of two sorted arrays.

Simultaneous Traversal:

  • Match: Check for duplicates (is it same as last added?). If unique, add to result. Increment BOTH pointers.
  • Mismatch: Increment the pointer of the smaller value to catch up.
Smallest Nonconstructible Value

Find the smallest positive integer that cannot be formed as the sum of a subset.

  1. Sort: Ascending order.
  2. Invariant: max_constructible (starts at 0).
  3. Gapping: If current_coin > max_constructible + 1, we have gapped. The answer is max_constructible + 1.
  4. Expanding: Otherwise, max_constructible += current_coin.
4. Sorting Theory
Concepts

Stable Sorting: Preserves the relative order of elements with equal keys.

Radix Sort: Sorts integers by processing individual digits into buckets. Often linear time.

Heap Basics: Max-heap gives O(log n) access to largest item. Useful for priority queues.
